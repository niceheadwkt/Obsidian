# AI 第二大腦與 Claude Cowork 自動化

**AI 第二大腦與自動化** 是一種將個人筆記系統（如 [[個人知識管理系統構築|Obsidian]]）與 AI 代理工具（如 [[Claude]] Desktop 及其 Cowork 功能）相結合的知識管理哲學。藉由設定自動化工作流，人負責「收集資訊」，AI 負責「整理提煉與定時分發」，從而實現個人與團隊的高效資訊運作。

---

## 1. 搭建 AI 第二大腦的核心架構

根據 Martina在進化 的實戰指南，非技術背景使用者亦可遵循以下工具鏈搭建「第二大腦」：

### 核心工具鏈
*   **知識存儲庫：Obsidian**
    *   以本地 Markdown 檔案為核心，支持雙向連結，可建立網狀知識圖譜。
    *   可搭配各類免費同步方案（例如 Git 版本控制），省去官方每月 4 美元的 Sync 費用。
*   **AI Agent 助手選擇**
    *   **[[Claude]] Code CLI**：適用於需要深入代碼庫、執行終端機命令、自主編譯與 Debug 的技術/半技術人員。
    *   **Hermes**：較輕量，適合作為非技術背景使用者的日常筆記整理與提煉助手。
*   **資訊擷取端：Web Clipper**
    *   透過瀏覽器剪輯插件，將網絡文章快速收錄至本地 Obsidian。
    *   實踐「我負責撿（收集），AI 負責整理（編譯）」的黃金協作流。
*   **靈感挖掘系統**
    *   利用自定義的斜槓命令（如 `/graduates`），自動掃描日記或隨手筆記，挖掘出有潛力、值得深挖的點子與專案線索。

## 2. Claude Cowork 自動化發日報實務

**Claude Cowork** 是 Claude 桌面 App 提供的強大工作流引擎，支持連接第三方服務（Connectors）並建立定時任務（Scheduled Tasks）。以下是每日定時篩選 AI 新聞並自動生成 Gmail 日報草稿的完整實作步驟：

### 前置準備
1.  必須為 **Claude Pro** 或 **Team** 付費帳戶。
2.  安裝官方 **Claude 桌面應用程式 (Desktop App)**。
3.  進入 Claude App 的 `Settings` $\rightarrow$ `Connectors` $\rightarrow$ 連結 **Gmail** 帳戶。

### 實作五步驟
1.  **授權連線**：在設定中完成 Gmail 連線授權，讓 Claude 具備將日報寫入 Gmail 草稿匣的權限。
2.  **建立專案工作資料夾**：
    *   在電腦本地建立一個獨立資料夾（例如：`AI 日報`）。
    *   在 Claude App 中點擊 `Projects` $\rightarrow$ `New Project`，並選擇該已存在的資料夾。此後所有日報的素材與歷史記錄都將存放於此。
3.  **初始化 `seen_urls.txt` 檔案**：
    *   在該專案對話框中向 Claude 下達指令：
        > 「在這個 Cowork 專案的工作資料夾裡，建立一個空白的 seen_urls.txt 檔案。內容留白即可，不需要寫任何東西進去。」
    *   此檔案將作為已發送新聞的「記憶庫」，避免連續數日推薦重複的新聞。
4.  **設定排程任務 (Scheduled)**：
    *   在專案右側面板點擊 `Scheduled` 建立定時任務。
    *   設定執行週期與時間（例如：每早 7:00）。
    *   填入任務指令 Prompt（參見下方實戰 Prompt 範本）。
5.  **手動測試 (Run Now)**：
    *   點擊 `Run now` 手動執行一次。
    *   **檢查指標 A**：Gmail 草稿匣是否成功出現主旨為 `📡 AI 產業動態 [YYYY-MM-DD]` 的草稿。
    *   **檢查指標 B**：專案資料夾下的 `seen_urls.txt` 是否已被寫入本次摘錄的 URL 記錄。

## 3. 實戰排程 Prompt 範本

在設定定時任務時，可使用以下繁體中文 Prompt 進行微調：

```markdown
讀取專案資料夾裡的 seen_urls.txt，略過所有已記錄的 URL。瀏覽以下來源，找出今天或昨天發布的 AI 產業動態：
1. https://www.bnext.com.tw（數位時代）
2. https://www.managertoday.com.tw（經理人）
3. 其他你覺得重要的來源

只要：大公司動作、新模型發布、產業策略、法規政策。
略過：工具教學、個人觀點、無新資訊的評論。
相同事件只保留一則，去重後 5–8 則。

用繁體中文整理成以下格式：
📡 [YYYY-MM-DD] AI 產業動態
🔥 今日重點（1–2 則）
[公司/事件]：[一句話說為什麼重要] 來源：[連結]
📋 其他動態（3–5 則）
[公司/事件]：[兩行以內] 來源：[連結]

存成 Gmail 草稿，主旨：📡 AI 產業動態 [YYYY-MM-DD]，收件人：[填入您的團隊 email]。
最後將本次所有 URL 追加到 seen_urls.txt，格式：[YYYY-MM-DD] 標題 | URL。
```

## 4. 關鍵防坑指南 (Troubleshooting)

在佈署自動化日報系統時，必須注意以下三個潛在限制，否則工作流將會失效：

1.  **篩選 Prompt 精細度**：對來源網站的指定以及「只要/略過」篩選標準必須定義得極其明確，這直接決定了 AI 篩選日報的最終品質。
2.  **必須有 `seen_urls.txt` 檔案**：如果沒有此防重檔案，AI 在每次排程執行時都將失去昨天的記憶，導致相同的新聞連續幾天重複出現在團隊日報中。
3.  **運行環境限制**：Claude Desktop App 的自動定時排程**不是運行在雲端伺服器**，而是本地代理。因此，在設定的排程時間點，**電腦必須處於開機狀態，且 Claude Desktop 程式必須保持開啟**，定時任務才會自動執行。
