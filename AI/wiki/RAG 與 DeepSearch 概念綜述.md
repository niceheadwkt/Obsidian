# RAG 與 DeepSearch 概念綜述 (Semantic Search & RAG Overview)

本頁面系統性探討了當前 AI 知識庫檢索與問答的核心技術：**RAG（檢索增強生成）** 與 **DeepSearch（深度搜尋）**。這兩項技術相輔相成，共同解決了傳統大語言模型 (LLM) 的資訊滯後與「AI 幻覺」痛點。

---

## 1. 深度搜尋 (DeepSearch)

**DeepSearch** 代表了搜尋引擎從「關鍵字比對」到「語意意圖理解」的技術演進。

- **核心技術支柱**：
  - **語意理解 (Semantic Understanding)**：基於 Transformer 架構，分析句子的脈絡、因果關係與語意。
  - **向量檢索 (Vector Search)**：利用 Embedding 技術將文章與問題轉化為多維數學空間的坐標。語意相似的文本，在向量空間中的距離就越近。
  - **重排機制 (Reranking)**：首階段粗篩出數百條相關資料後，使用更精密的 AI 模型進行二次語意排序，確保最精準的內容排在最前。
  - **多模態搜尋 (Multimodal Search)**：支援文字、圖片、影片、音訊的交叉檢索。
- **傳統關鍵字搜尋 vs. DeepSearch**：

| 比較維度 | 傳統關鍵字搜尋 (BM25) | 深度語意搜尋 (DeepSearch) |
| :--- | :--- | :--- |
| **核心機制** | 檢查文字中是否包含相同的單字 | 理解詞彙在情境中的**真正含義** |
| **錯字與同義詞** | 容易因錯字或用詞不同而找不到結果 | 自動識別同義詞並校正搜尋意圖 |
| **複雜描述/長句** | 拆碎成單字，容易斷章取義 | 能處理具有故事性與因果關係的提問 |
| **代表實例** | 輸入「主角會忘記事情、手上記滿刺青的老電影」，傳統搜尋可能失焦；DeepSearch 能精確導向電影 **《記憶拼圖》(Memento)**。 |

## 2. 檢索增強生成 (RAG)

**RAG** 是一種結合「外部知識庫檢索」與「LLM 生成」的架構，好比讓 AI 在回答問題時可以 **「開卷查資料」**。

### 核心運作三步驟
```
[使用者提問] ──> 1. 檢索 (Retrieval) ──> 2. 增強 (Augmentation) ──> 3. 生成 (Generation) ──> [精確回答]
                        │                               ▲
                        ▼                               │
                  [向量資料庫] ─────────────────────────┘
```
1. **檢索 (Retrieval)**：將問題向量化，在向量資料庫中搜尋最相關的文字片段（如 PDF 段落、網頁等）。
2. **增強 (Augmentation)**：將檢索出的相關資料拼接至原始問題中，組成富含事實上下文的 Prompt。
3. **生成 (Generation)**：LLM 閱讀這些參考資料，整理並輸出有事實根據的回答，並標明出處。

### RAG 的核心優勢
- **消除 AI 幻覺**：強制 LLM 僅能根據提供的資料回答，否則回答「不知道」。
- **使用私有/即時資料**：無須耗資重新微調 (Fine-tuning) 模型，即可讓 AI 掌握企業內部機密或即時資訊。
- **高性價比**：建立與維護向量資料庫（如 Chroma, Pinecone, Qdrant）的成本遠低於微調模型。

---

## 3. 當前應用與生態系

1. **AI 搜尋引擎**（如 Perplexity, Gemini Deep Research）：當開啟深度搜尋時，AI 會主動拆解問題、多角度檢索數十篇文獻，整合產出結構化報告。
2. **企業私有知識庫**：工程師或分析師可用自然語言查詢幾十萬頁的代碼、ERP 流程與規格書。
3. **主流 RAG 技術棧**：
   - **編排框架**：LangChain、LlamaIndex。
   - **向量資料庫**：Chroma、Pinecone、Milvus、PGVector。
   - **應用前端**：Streamlit、Chainlit。
   - **本地運行引擎**：透過 [[Ollama]] 驅動本地開源模型（如 Qwen3），實現完全零 Token 成本的本地知識庫。
