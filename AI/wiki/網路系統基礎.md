# 網路與企業系統基礎 (Networking & Enterprise Systems)

本頁面彙整了區域網路 (LAN) 與廣域網路 (WAN) 的底層通訊機制、監視器影音串流的調試實務、以及企業級 **DPMS** 開發框架的核心定位。

---

## 1. IP 位址與 MAC 位址解析 (ARP 協定)

在網路通訊中，將已知 IP 位址轉化為硬體 MAC 位址的核心機制是 **ARP (位址解析協定, Address Resolution Protocol)**。

### 查詢方法
- **命令列查詢 (暫存表)**：
  - Windows CMD/PowerShell: `arp -a [目標IP]`
  - Linux/macOS Terminal: `arp -n [目標IP]`
- **強制更新 ARP 暫存 (Ping 法)**：若暫存失效，可先執行 `ping [目標IP]`，再執行 `arp -a` 查詢。
- **區網掃描**：使用 `Nmap` 指令 `sudo nmap -sn 192.168.1.0/24` 掃描整個網段的 IP/MAC 對應。
- **Python 自動化**：利用 `scapy` 封包建構庫送出 ARP 廣播請求：
  ```python
  from scapy.all import ARP, Ether, srp
  # 送出 ARP Request 廣播，回收硬體 hwsrc 即可取得 MAC
  ```

### ⚠️ 技術邊界與限制
- **僅限區域網路 (LAN)**：ARP 僅能在同一個二層子網內運作。您**無法**跨越路由器、防火牆或在 Internet 上直接取得某個外網 IP 的 MAC 位址。
- **外網識別替代方案**：若要在公網識別裝置身份，應改用 **Browser Fingerprinting（瀏覽器指紋）**、**Cookie 寫入 UUID** 或行動端的 **Instance ID**，而非依賴 MAC。

## 2. 監視器 RTSP 影音串流調試實務

以已知外部 IP `211.78.6.68` 與 MAC `00:0E:53:36:3B:B3` 取得監視器畫面為例，核心調試流程如下：

### 步驟 A：識別設備與埠口 (Ports)
1. **廠商識別**：透過 MAC 前綴 `00:0E:53` 查詢，該設備屬於 **GKB (全視界)** 或其代工廠，可藉此查詢其預設之管理帳密。
2. **通訊埠定位**：Web 管理通常使用 Port `80` 或 `8080`；影音串流（RTSP）預設使用 Port `554`。

### 步驟 B：取得 RTSP 串流畫面
RTSP (Real-Time Streaming Protocol) 是多數網路攝影機 (IP Cam) 支援的串流協定。
- **串流網址格式範例**：`rtsp://admin:password@211.78.6.68:554/live/ch0`
- **播放方法**：
  1. **VLC 播放器 (最簡單)**：開啟 VLC $\rightarrow$ 點選「媒體」 $\rightarrow$ 「開啟網路串流」 $\rightarrow$ 貼上 RTSP 網址即可播放。
  2. **Python OpenCV (適合影像辨識/AI 分析)**：
     ```python
     import cv2
     cap = cv2.VideoCapture("rtsp://admin:password@211.78.6.68:554/live/ch0")
     # 透過循環 cap.read() 即可將影像幀（Frame）實時印出與分析
     ```
  3. **手機端 App**：使用 *IP Cam Viewer* 等應用程式手動配置 IP 與 RTSP TCP/UDP 協定。
- **常見障礙**：外網連線時，前端的路由器必須做好 **Port Forwarding (虛擬伺服器將 554 埠轉發至監視器內網 IP)**，否則封包會被防火牆阻擋。

## 3. DPMS 企業級開發框架

在企業應用中，**DPMS (Distributed Project Management System，分佈式專案管理系統)** 通常指一套針對 Java/JSP 大型 ERP 系統所設計的模組化開發框架（如 ICSC DPMS）。

- **系統特性**：
  - **模組化維護**：將複雜的商業邏輯（包含後端 DB2 資料庫操作、大規模 SQL 報表）拆解為重用組件。
  - **程式碼規範化**：提供統一的 JSP 與 Java 寫作模板，以便開發團隊協作並降低維護 Legacy Code（遺留系統）的難度。
- **技術挑戰**：
  - 進行 DB2 資料庫效能調優（優化複雜的 SQL JOIN 查詢）。
  - 將舊有的 JSP 業務邏輯與現代自動化工具（如 Docker 容器化、Python 數據提取腳本、MCP 協議）進行安全對接。
