# 前端與系統開發常用技術指南 (Developer Tools & Best Practices)

本頁面彙整了專案開發中常見的工具使用、核心機制與最佳實踐，涵蓋版本控制 (Git)、跨平台框架 (Electron)、Unix 腳本執行 (Shebang)、以及網路分發 (CDN)。

---

## 1. 版本控制系統：Git 新手實戰

**Git** 是一個分散式版本控制系統，如同專案的「時光機」與「儲存分身」。

### 核心工作流
```
[工作目錄 (Working Dir)] ──(git add)──> [暫存區 (Staging Area)] ──(git commit)──> [本地儲存庫 (Local Repo)] ──(git push)──> [遠端 GitHub (Remote)]
```

- **第一步：配置身份**（一生一次）：
  ```bash
  git config --global user.name "您的名字"
  git config --global user.email "your_email@example.com"
  ```
- **第二步：初始化**：於資料夾內執行 `git init`，會建立一個隱藏的 `.git` 資料夾作為版本資料庫。
- **第三步：暫存與提交**：
  - 查看狀態：`git status`
  - 暫存變更：`git add .`（句點代表全部）
  - 提交存檔：`git commit -m "本次修改說明的備忘錄"`
- **第四步：分支管理 (Branching)**：
  - 建立分支：`git branch feature-login`
  - 切換分支：`git checkout feature-login`
  - 合併回主線：切回主線 `git checkout main` 後，執行 `git merge feature-login`。
- **第五步：防呆救援指令**：
  - 放棄單一檔案的未存檔修改：`git checkout -- <檔名>`
  - 強制退回到上一次 commit 狀態：`git reset --hard HEAD`（慎用）

## 2. 跨平台開發與環境規範

### Electron 框架
- **定位**：使用 HTML、CSS 和 JavaScript 構建跨平台桌面應用程式的開源框架。
- **原理**：將 Chromium 瀏覽器引擎與 Node.js 運行環境相結合，使網頁應用能直接擁有作業系統級別的存取權限（如讀寫本地檔案、調用 API）。

### Shebang (#!) 規範
- **定義**：在 Unix-like 系統（Linux, macOS）的腳本檔案第一行開頭的字元序列。
- **作用**：告訴系統應使用哪一個直譯器來執行該腳本。
- **最佳實踐**：
  - 執行 Bash 腳本：`#!/bin/bash` 或 `#!/usr/bin/env bash`
  - 執行 Node.js 腳本：`#!/usr/bin/env node`
  - 執行 Python 腳本：`#!/usr/bin/env python3`
  - *註*：使用 `/usr/bin/env` 的好處是它會自動在使用者的系統環境變數 (`$PATH`) 中尋找直譯器，具備更好的跨平台移植性。

## 3. 專案輔助與網路分發

### CDN (內容傳遞網路)
- **原理**：在遍佈全球的伺服器節點上快取網站的靜態資源（如 JS, CSS, 圖片）。
- **作用**：當使用者造訪網站時，系統會自動導向距離最近的 CDN 節點，大幅減少伺服器負載並加速網頁載入速度。

### Snippets (程式碼片段)
- **定位**：預先定義好的常用代碼範本，通常可在編輯器（如 VS Code）中輸入簡寫並按下 `Tab` 鍵快速展開，可大幅提升編碼效率。

### Slack 整合
- **應用**：利用 Webhooks 將系統警報、自動部署狀態或 Git 提交日誌自動推送至特定 Slack 頻道，實現團隊協作自動化。
